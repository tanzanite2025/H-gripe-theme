## 🎨 设计规范

### 统一的代码风格
所有页面遵循相同的设计模式：

1. **页面结构**
   - 页面头部（标题 + 操作按钮）
   - 统计卡片（可选）
   - 筛选表单
   - 数据表格
   - 分页组件

2. **组件使用**
   - Element Plus UI 组件库
   - 响应式布局
   - 统一的颜色方案
   - 一致的间距和字体

3. **交互模式**
   - 对话框用于创建/编辑
   - 确认框用于删除操作
   - 消息提示用于操作反馈
   - 加载状态显示

4. **权限控制**
   - 使用 `hasPermission()` 方法
   - 按钮级别的权限控制
   - 与后端权限系统对应

## 🔧 技术实现

### 核心技术栈
- **Vue 3**: Composition API
- **Element Plus**: UI 组件库
- **Axios**: HTTP 客户端
- **Pinia**: 状态管理（auth store）

### 关键功能
1. **数据获取**: 使用 axios 调用后端 API
2. **表单验证**: Element Plus 表单验证规则
3. **权限控制**: 基于 auth store 的权限检查
4. **状态管理**: 响应式数据和计算属性
5. **错误处理**: 统一的错误提示

### API 集成
所有页面都正确集成了对应的后端 API：
- `GET /api/admin/faqs` - FAQ 列表
- `GET /api/admin/galleries` - 图库列表
- `GET /api/admin/subscriptions` - 订阅列表
- `GET /api/admin/tickets` - 工单列表
- `GET /api/admin/marketing/*` - 营销管理
- `GET /api/admin/settings/*` - 系统设置
- `GET /api/admin/logs` - 审计日志

### 开发环境运行
```bash
cd go-backend/web/admin
npm install
npm run dev
```

### 生产环境构建
```bash
npm run build
```

- 开发环境: `http://localhost:5173`
