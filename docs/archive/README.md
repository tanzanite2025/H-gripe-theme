# Documentation Archive

This directory stores historical project notes and completion reports.

Archived documents are not authoritative. They may mention old architecture, old API paths, old readiness claims, or implementation details that have since changed.

## Categories

- `project/` - old project status, completion, deployment, monitoring, payment, storage, and fix reports.
- `backend/` - old backend quality, security, frontend-page, and completion reports.
- `audit/` - old audit, optimization, and data-sync review reports.
- `refactoring/` - old handler refactoring reports.
- `bugfix/` - old bugfix reports.
- `splitting/` - old file-splitting reports.
- `optimization/` - old cleanup, optimization, and refactoring progress reports.

## Rules

- Do not link archived reports as current onboarding material.
- Do not treat archive claims as verified facts.
- If a historical note becomes useful again, rewrite it as a current guide outside `archive/`.
